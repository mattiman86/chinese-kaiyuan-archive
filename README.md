# chinese-kaiyuan-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容不限于某一特定域名或网站，而是涵盖各类与中文开源相关的信息、文档、工具或资源展示。

## 项目简介

chinese-kaiyuan-archive 是一个静态 HTML 页面集合仓库。每个页面独立存在，彼此不依赖，可直接通过浏览器打开查看。仓库主要目的是保存那些可能随时间变化或消失的网页内容，为中文开源社区提供一个备份与存档空间。

## 目录说明

```
.
├── pages/           # 存放所有已归档的 HTML 页面
├── assets/          # 页面引用的静态资源（如图片、样式、脚本）
└── README.md        # 本说明文件
```

- `pages/` 目录下每个子文件夹或文件代表一个独立的归档页面，命名尽量反映内容主题。
- `assets/` 目录用于存放页面所需的公共资源，避免页面内重复引用外部链接。
- 如有需要，可在 `pages/` 内按年份、主题或来源进一步组织子目录。

## 页面归档说明

每个归档页面都尽量保留原始内容的完整结构与样式。若原始页面包含外部资源（如字体、图片、脚本），已尽可能下载至 `assets/` 目录并更新引用路径，以确保离线可查看。

归档页面可能包含以下内容类型：
- 开源项目介绍或文档
- 社区讨论与公告
- 技术教程与指南
- 工具或资源列表

请注意：归档页面中的信息可能已过时，不代表当前状态。引用时请自行核实。

## 维护说明

本仓库由志愿者维护，欢迎提交 Issue 或 Pull Request 贡献新的归档页面。

### 贡献方式

1. 将准备归档的 HTML 页面保存至 `pages/` 目录下。
2. 如有外部资源，请下载至 `assets/` 目录并更新页面中的引用路径。
3. 确保页面可以独立打开，不依赖外部网络。
4. 提交 Pull Request，简要说明页面来源与归档原因。

### 注意事项

- 请勿归档包含恶意代码、侵犯版权或违反法律法规的内容。
- 尽量保留页面原始面貌，但可适当修正明显的显示问题（如乱码、断链）。
- 若页面体积过大，建议压缩或分割后提交。
